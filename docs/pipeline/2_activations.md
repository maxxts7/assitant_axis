# `pipeline/2_activations.py`

## Overview

This script is the second stage of the pipeline. It takes the model responses generated by stage 1 and **extracts internal neural network activations** (hidden states) from specified layers of the model. For each conversation, it computes the mean activation vector across assistant turns and saves the results as `.pt` (PyTorch tensor) files, one per role.

These activation vectors are the raw data used for downstream probing and analysis -- they capture what the model is "thinking" internally when it produces a response under a given role.

Key features:
- Supports **multi-worker parallelization** across GPUs (same pattern as stage 1).
- Skips roles whose activation files already exist (idempotent/restartable).
- Configurable layer selection: extract all layers or a specific subset.

```python
def load_responses(responses_file: Path) -> List[dict]:
    """Load responses from JSONL file."""
    responses = []
    with jsonlines.open(responses_file, 'r') as reader:
        for entry in reader:
            responses.append(entry)
    return responses
```

Reads a JSONL file (one JSON object per line) and returns a list of dictionaries. Each entry contains a conversation (list of messages), metadata like `prompt_index`, `question_index`, and a `label`.

#### Mapping Spans to Per-Turn Activations (Lines 100-102)

```python

```

Uses the `SpanMapper` to extract and average activations for each turn in each conversation. Returns a list of tensors, each of shape `(num_turns, num_layers, hidden_size)`.

#### Extracting Assistant Activations (Lines 104-124)

```python
        for conv_acts in conv_activations_list:
            if conv_acts.numel() == 0:
                all_activations.append(None)
                continue

            if conv_acts.shape[0] >= 2:
                assistant_act = conv_acts[1::2]  # All assistant turns
                if assistant_act.shape[0] > 0:
                    mean_act = assistant_act.mean(dim=0).cpu()  # (num_layers, hidden_size)
                    all_activations.append(mean_act)
                else:
                    all_activations.append(None)
            else:
                all_activations.append(None)
```

For each conversation's activations:
1. If the tensor is empty, append `None`.
2. Conversations alternate user/assistant turns (turn 0 = user, turn 1 = assistant, etc.). The slice `[1::2]` selects all odd-indexed turns (assistant turns).
3. If there are multiple assistant turns, their activations are averaged together with `.mean(dim=0)`.
4. The result is moved to CPU to free GPU memory.
5. The final shape is `(num_layers, hidden_size)` -- one activation vector per layer.

#### Memory Cleanup (Lines 126-129)

```python
        del batch_activations
        if (batch_start // batch_size) % 5 == 0:
            torch.cuda.empty_cache()
```

Deletes the batch activations tensor to free GPU memory. Every 5 batches, `torch.cuda.empty_cache()` is called to return freed GPU memory to the CUDA allocator (this is an expensive operation, so it is not done every batch).

#### Collecting Role Files (Lines 272-291)

    output_dir = Path(args.output_dir)

#### GPU Partitioning and Role Distribution (Lines 295-308)

```python
    gpu_chunks = []
    for i in range(num_workers):
        start = i * tensor_parallel_size
        end = start + tensor_parallel_size
        gpu_chunks.append(gpu_ids[start:end])

    role_chunks = [[] for _ in range(num_workers)]
    for i, role_file in enumerate(role_files):
        role_chunks[i % num_workers].append(role_file)

    for i in range(num_workers):
        logger.info(f"Worker {i} (GPUs {gpu_chunks[i]}): {len(role_chunks[i])} roles")
```

Partitions GPUs into contiguous chunks. Roles are distributed in a round-robin fashion (role 0 goes to worker 0, role 1 to worker 1, etc., wrapping around). This is a slight difference from stage 1, which uses contiguous chunks.

#### Spawning Processes (Lines 310-330)

```python
    mp.set_start_method('spawn', force=True)

    processes = []
    for worker_id in range(num_workers):
        if role_chunks[worker_id]:
            p = mp.Process(
                target=process_roles_on_worker,
                args=(worker_id, gpu_chunks[worker_id], role_chunks[worker_id], args)
            )
            p.start()
            processes.append(p)

    logger.info(f"Launched {len(processes)} worker processes")
    for p in processes:
        p.join()

    logger.info("Multi-worker processing completed!")
    return 0
```

Uses `spawn` start method (required for CUDA). Launches one process per worker, then waits for all to finish.
